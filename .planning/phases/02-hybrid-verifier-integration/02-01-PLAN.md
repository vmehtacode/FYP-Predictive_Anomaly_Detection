Create `src/fyp/selfplay/hybrid_verifier.py` and implement the `PhysicsConstraintLayer` class and the `tolerance_band_score()` helper function.

**`tolerance_band_score(value, lower_safe, upper_safe, lower_limit, upper_limit) -> float`** (module-level function):
- Returns 0.0 if value is in [lower_safe, upper_safe] (safe zone)
- Returns 1.0 if value is below lower_limit or above upper_limit (full violation)
- Returns linear interpolation in warning zones: (lower_safe - value) / (lower_safe - lower_limit) for lower warning, (value - upper_safe) / (upper_limit - upper_safe) for upper warning
- See 02-RESEARCH.md Pattern 2 for exact implementation

**`PhysicsConstraintLayer`**:
```python
class PhysicsConstraintLayer:
    def __init__(self, config: PhysicsConfig) -> None:
        # Store config, pre-compute absolute voltage limits from percentages
        # e.g., lower_limit = nominal_v * (1 + lower_limit_pct/100)

    def evaluate(self, forecast: np.ndarray) -> tuple[np.ndarray, dict]:
        # Returns (per-node severity scores [0,1], details dict)
        # For each node:
        #   1. voltage_score = tolerance_band_score(forecast_value, voltage bounds)
        #   2. capacity_score = tolerance_band_score(abs(forecast_value), capacity bounds)
        #   3. ramp_score = compute ramp rate from consecutive values, tolerance_band_score
        # Combine per-constraint scores: max(voltage, capacity, ramp) per node
        # Per user decision: continuous severity scores, NOT binary flags
```

For **voltage scoring**: Convert config percentages to absolute values (e.g., lower_limit = 230 * (1 + (-10/100)) = 207V). Apply tolerance_band_score per node.

For **capacity scoring**: Use typical_max_kw as upper_safe, absolute_max_kw as upper_limit, with overload_threshold_pct and critical_threshold_pct defining the warning zone boundaries. Lower bounds are 0 (non-negative).

For **ramp rate scoring**: Compute differences between consecutive forecast values. Use warning_ramp as upper_safe, max_ramp as upper_limit. Lower bounds mirror upper.

The details dict should contain per-constraint scores for interpretability: `{"voltage_scores": array, "capacity_scores": array, "ramp_scores": array, "combined_scores": array}`.

Use `from __future__ import annotations`. Import numpy as np. Import PhysicsConfig from hybrid_verifier_config.
